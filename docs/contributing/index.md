# Contributing

Guidelines for contributing to the Aincrad platform infrastructure.

## Who Should Read This

This section is for:
- Platform engineers making changes to organizational infrastructure
- Contributors adding features or fixing issues
- Anyone documenting or improving the platform

## Overview

The platform uses infrastructure as code to manage GCP organization structure, projects, and shared services. All changes go through code review and testing before deployment.

## Sections

- [Setup](setup.md) - Get started with local development environment
- [Usage](usage.md) - Day-to-day development commands and workflows
- [Workflows](workflows.md) - CI/CD pipeline and automation processes
- [Contributing](contributing.md) - Code review process, commit conventions, and testing requirements
- [Documentation](documentation.md) - Writing and maintaining documentation

## Quick Start

1. Read [Setup](setup.md) to configure your local environment
2. Review [Contributing](contributing.md) for commit message format and review process
3. Use [Usage](usage.md) for common development tasks
4. Follow [Documentation](documentation.md) when updating docs
